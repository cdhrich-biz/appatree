# 원격 가족 지원 베타 테스트 체크리스트

APPATREE 원격 가족 지원 기능의 실사용 검증용 가이드. 두 개의 브라우저 프로필(또는 데스크톱+모바일)을 "부모"와 "자녀" 계정으로 로그인해서 한 사람이 전체 흐름을 확인할 수 있다.

## 0. 사전 준비

- `.env.local`에 `ABLY_API_KEY` 등록 (`xxx.yyy:zzz` 형식)
- (선택) `METERED_TURN_KEY`/`METERED_TURN_DOMAIN` — 모바일 셀룰러에서 음성 통화를 테스트할 때 필요
- Supabase DB에 마이그레이션 적용 (MCP `apply_migration` 또는 `pnpm db:push`)
- 서버 실행: `pnpm dev` (또는 `npm run dev`)

## 1. 기초 연결 플로우

| # | 시나리오 | 기대 결과 |
|---|---|---|
| 1 | 부모 브라우저로 로그인 → 홈의 **"도움 요청"** 버튼 클릭 | `/help` 화면에 6자리 코드 표시 + 음성으로 "여섯 자리 숫자가 보이시나요?" TTS 재생 |
| 2 | 자녀 브라우저로 로그인 → 설정 → **"부모님 연결"** → 6칸 입력 | 5자리까지 입력해도 자동제출 안 됨, 6자리 되면 자동으로 연결 성공 애니메이션 표시 |
| 3 | 잘못된 코드 6자리 입력 | 토스트 "유효하지 않은 코드입니다" + 입력칸 초기화, 자동 재포커스 |
| 4 | 동일 코드를 자녀 두 명이 동시에 입력 | 한 명만 성공, 다른 한 명은 "이미 사용된 코드" 에러 |
| 5 | 10분 지나서 코드 사용 | "만료된 코드" 에러 |
| 6 | 자녀 **`/family`** 로 이동 | 연결된 부모님 카드 + "도와드리기" 버튼 보임 |

## 2. 세션 요청 & 수락

| # | 시나리오 | 기대 결과 |
|---|---|---|
| 1 | 자녀 "도와드리기" 클릭 | 자녀: `/remote/<key>`로 이동, "부모님 응답을 기다리는 중" 표시 |
| 2 | 부모 화면 상태 확인 | 풀스크린 다이얼로그 + "수락" (녹색) / "거절" 두 큰 버튼 + TTS 자동 안내 + 45초 카운트다운 |
| 3 | 45초 방치 | 자동으로 거절 처리, 자녀 화면은 "거절되었어요" 후 3초 뒤 `/family` |
| 4 | 부모 "수락" 클릭 | 자녀 화면이 "active"로 전환, 부모는 상단에 녹색 배너 "xxx가 도와주는 중" |
| 5 | 부모 "거절" 클릭 | 자녀 화면 "거절되었어요" |

## 3. 원격 조작 & 미러링

| # | 시나리오 | 기대 결과 |
|---|---|---|
| 1 | 자녀 화면의 미러 iframe | 부모가 보는 현재 route가 observer 모드로 렌더됨 (인터랙션 차단) |
| 2 | 부모가 `/library`로 이동 | 자녀 iframe도 자동으로 `/library` 로 업데이트 |
| 3 | 자녀 "검색 도와드리기" 에 "트로트" 입력 → 검색 | 부모 앱이 `/search?q=트로트` 로 이동 + 토스트 "자녀가 '트로트' 검색을 도와드려요" |
| 4 | 자녀 "음성 안내"에 "엄마 여기 눌러봐" 입력 → 전달 | 부모 기기에서 해당 문장 TTS 재생 |
| 5 | 자녀 "여기 누르세요 표시" 모드 켜기 → iframe 안의 버튼 클릭 | 부모 화면의 같은 버튼에 주황 펄스 테두리 5초 표시 + scrollIntoView |
| 6 | 부모 `/admin`으로 이동하는 경로를 자녀가 전송 시도 | `action:blocked` 수신, 자녀에게 "허용되지 않은 경로" 토스트 |

## 6. 종료 & 오류

| # | 시나리오 | 기대 결과 |
|---|---|---|
| 1 | 부모 "종료" 버튼 | 자녀 화면 "도와주기가 끝났어요" + TTS + 3초 후 `/family` |
| 2 | 자녀 세션 중 뒤로가기 | 자동 `endSession` 호출, 부모 배너 사라짐 |
| 3 | Ably disconnected 상태 | 중앙 상단에 주황 "연결이 잠시 끊겼어요" 배너 |
| 4 | Ably failed | 빨강 "새로고침이 필요해요" + 새로고침 버튼 |
| 5 | 부모가 `/admin` 경로에 있을 때 자녀가 원격 접속 | 자녀는 requestSession 성공, 세션은 active지만 자녀가 원격 조작해도 차단됨 |

## 7. 감사 로그

- Supabase 대시보드에서 `remote_actions` 테이블 확인
- 한 세션에서 실행된 모든 `action:*` 이벤트가 행으로 남아야 함
- `payload` 칼럼에 JSON 문자열로 route/query/videoId 등이 저장됨

## 9. 알려진 제약

- **Service Worker 킬스위치 상태** — 푸시 알림 수신은 동작하지 않음. 부모가 앱 탭에 머무는 동안만 요청 감지 (Phase 2에서 FCM 추가 예정)
- **iOS Safari 멀티탭** — iframe 미러는 동작, `getDisplayMedia`는 미지원이지만 우리 구현은 state:sync 방식이라 영향 없음
- **TURN 미구성** — 엄격 NAT/셀룰러 환경에서 음성이 연결되지 않을 수 있음
- **세션 복구** — 자녀/부모가 브라우저를 새로고침하면 활성 세션 state가 사라짐 (localStorage 저장 미구현, Phase 2 과제)
- **자녀 관리자 기능 차단** — 자녀는 부모의 `/admin`을 열 수 없음 (화이트리스트)

## 10. Vercel 배포 체크리스트

1. `vercel env add ABLY_API_KEY` (Production + Preview)
2. `vercel env add METERED_TURN_KEY` `METERED_TURN_DOMAIN` (선택)
3. `DATABASE_URL` / `JWT_SECRET` / `OAUTH_SERVER_URL` / `OWNER_OPEN_ID` / `VITE_APP_ID` / `BUILT_IN_FORGE_API_KEY` / `YOUTUBE_API_KEY` 이미 설정되어 있는지 확인
4. `vercel deploy --prod`
5. 배포 후 프로덕션 URL에서 위 시나리오 1~9 재검증
6. Ably 대시보드 Usage 탭에서 동접/메시지 수 모니터링
