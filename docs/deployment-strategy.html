APPATREE 배포 전략

v1.0.0

1 개요

APPATREE는 React 19 + Vite 프론트엔드와 Express.js + tRPC 백엔드로 구성된 풀스택 모노리스 앱이다. 배포 시 프론트엔드(SPA)와 백엔드(API 서버)를 함께 서빙해야 한다.

React 19 Vite 7 Express.js tRPC MySQL Tailwind CSS 4

2 플랫폼 비교: Vercel vs Cloudflare

항목 Vercel Cloudflare
Express.js 호환 @vercel/node 어댑터로 가능 불가 (V8 isolate)
MySQL 연결 Serverless pooling 필요 Hyperdrive 필요, mysql2 호환 불안정
한국 리전 서울 엣지 노드 서울 포함 300+ 엣지
Cold Start 250-500ms ~5ms (Workers)
리팩터링 비용 1-2일 2-4주 (전면 재작성)
무료 티어 100GB 대역폭, 100GB-hrs 100K req/day, 무제한 정적
PWA 지원 완전 지원 완전 지원
WebSocket 미지원 (서버리스) Durable Objects
빌드 파이프라인 Vite 네이티브 지원 Pages 정적만, Workers 별도
환경 변수 대시보드 UI, 환경별 분리 Wrangler CLI / 대시보드

Vercel

장점

  • + @vercel/node 어댑터로 Express.js 래핑 가능
  • + Vite 빌드 네이티브 지원
  • + 서울 엣지 노드, 빠른 정적 자산 전달
  • + Git push 기반 자동 배포 + Preview
  • + 우수한 모니터링/분석 대시보드

단점

  • - MySQL 커넥션 관리 복잡
  • - 실행 시간 제한 (Hobby 10초 / Pro 60초)
  • - WebSocket 미지원
  • - Cold Start 250-500ms

필요한 코드 변경

  1. api/index.ts 생성 (Express 앱 export)
  2. vercel.json 생성 (라우팅 설정)
  3. server/db.ts connection pool 적용
  4. package.json vercel-build 추가

Cloudflare

장점

  • + Workers near-zero cold start (~5ms)
  • + 300+ 글로벌 엣지, 한국 내 다수 PoP
  • + Pages 무료 무제한 정적 대역폭
  • + Durable Objects로 WebSocket 지원

단점

  • - Express.js 실행 불가 (V8 isolate)
  • - mysql2 드라이버 호환 불가
  • - 백엔드 전면 재작성 필요
  • - D1은 SQLite만, MySQL은 Hyperdrive
  • - CPU 시간 제한 (무료 10ms / 유료 30초)

필요한 코드 변경

  1. Express → Hono 또는 itty-router
  2. mysql2 → Hyperdrive 호환 드라이버
  3. Node.js 빌트인 → Web API 대체
  4. axios → fetch 전환
  5. tRPC Express → Fetch adapter
  6. 예상 소요: 2-4주

3 권장 배포 전략

Option A

최우선 추천: Vercel

현재 아키텍처에서 가장 적은 변경으로 배포 가능한 플랫폼.

[사용자] → [Vercel Edge (서울)]
           ├── 정적 자산 (dist/public/) → CDN 캐시
           └── /api/* → Serverless Function (Express.js)
                         └── MySQL (외부 DB 서비스)
Option B

최적 추천: 하이브리드 (VPS + CDN)

BEST

Express.js를 수정 없이 배포하고 CDN으로 성능 최적화.

[사용자] → [Cloudflare CDN]
           ├── 정적 자산 → 엣지 캐시 (< 50ms)
           └── /api/* → [Fly.io / Railway 서울/도쿄]
                         ├── Express.js (상시 실행)
                         └── MySQL (persistent connection)
코드 변경 불필요
Cold start 없음
MySQL 커넥션 풀링 안정적
정적 자산 < 50ms 응답
향후 WebSocket 추가 용이

추천 VPS 플랫폼

플랫폼 한국 근접 리전 무료 티어 유료 특징
Fly.io 서울(ICN), 도쿄 3 shared VM ~$5-7/월 서울 직접 배포
Railway 도쿄 $5 크레딧/월 ~$5-10/월 Zero-config, MySQL 애드온
Render 싱가포르 750시간/월 $7/월 간단, Git 자동 배포

4 환경 변수 설정 가이드

변수명 설명 예시
VITE_APP_ID 애플리케이션 ID appatree
JWT_SECRET 쿠키 서명 시크릿 your-secret-key
DATABASE_URL MySQL 연결 문자열 mysql://user:pass@host:3306/db
OAUTH_SERVER_URL OAuth 서버 URL https://oauth.example.com
OWNER_OPEN_ID 관리자 OAuth ID admin-open-id
NODE_ENV 실행 환경 production
BUILT_IN_FORGE_API_URL Forge API 엔드포인트 https://forge.example.com
BUILT_IN_FORGE_API_KEY Forge API 키 forge-api-key
VITE_OAUTH_PORTAL_URL OAuth 포털 URL https://portal.example.com
